# CalcPro2.1

## Восстановление после потери ПК

Этот проект теперь можно скачать из репозитория и запустить без обязательного локального `.env`:

```bash
npm install
npm run dev
```

Почему это работает:

- Firebase Web config уже зашит как fallback в клиентский код.
- `.env` нужен только если вы хотите локально переопределить Firebase-проект или дополнительные `VITE_*` настройки.
- Настоящие серверные секреты нельзя хранить только в публичном frontend-репозитории. Если позже появятся private API keys, их нужно выносить в backend / Cloud Functions или хранить отдельно в защищённом backup.

## Local запуск

1. Установить зависимости:

```bash
npm install
```

2. Запустить dev-сервер (обычный):

```bash
npm run dev
```

3. Или запустить на фиксированном адресе `127.0.0.1:5173`:

```bash
npm run dev:local
```

После запуска откройте URL из терминала (обычно `http://127.0.0.1:5173/` или `http://localhost:5173/`).

## Чтобы работала авторизация и все Firebase-функции

- `.env` для базового запуска больше не обязателен.
- Если хотите переопределить Firebase-конфиг локально, создайте `.env` из `.env.example` и заполните `VITE_FIREBASE_*`.
- В Firebase Console → Authentication → Settings → Authorized domains добавьте `localhost`.
- Если используете `VITE_ALLOWED_HOSTS`, убедитесь что там есть `localhost,127.0.0.1`.
- После изменений `.env` перезапустите dev-сервер.
